采用 hexo （博客框架） + github page （存放静态页面） + github Actions（自动部署） 技术搭建而成。
主题采用next主题。

需要创建一个github的仓库，里面有两个分支，一个用于放源码，一个放编译后的静态页面

比如： master ：存放静态页面 , src : 存放源码

自动部署脚本: 

```
name: Blog CI/CD

# 触发条件：在 push 到 src 分支后触发
on:
  push:
    branches:
      - src
#运行系统
jobs:
  build:
    runs-on: ubuntu-latest
#nodejs版本
    strategy:
      matrix:
        node-version: [14.x, 15.x,16.x,17.x]

    steps:
      - name: Checkout codes
        uses: actions/checkout@v2

      - name: Setup node
        # 设置 node.js 环境
        uses: actions/setup-node@v1
        with:
          node-version: ${{ matrix.node-version }}

      - name: Cache node modules
        # 设置包缓存目录，避免每次下载
        uses: actions/cache@v2
        with:
          path: ~/.npm
          key: ${{ runner.os }}-node-${{ hashFiles('**/package-lock.json') }}

      - name: Install hexo dependencies
        # 下载 hexo-cli 脚手架及相关安装包
        run: |
          npm install -g hexo-cli
          npm install
      - name: Generate files
        # 编译 markdown 文件
        run: |
          hexo clean
          hexo generate
      - name: Deploy hexo blog
        env:
          # Github 仓库
          GITHUB_REPO: github.com/dawnflyc/dawnflyc.github.io.git
        # 将编译后的博客文件推送到指定仓库
        run: |
          cd ./public && git init && git add .
          git config user.name "dawnflyc"
          git config user.email "dawnflyc@gmail.com"
          git add .
          git commit -m "GitHub Actions Auto Builder at $(date +'%Y-%m-%d %H:%M:%S')"
          git push --force --quiet "https://${{ secrets.GIT_TOKEN }}@$GITHUB_REPO" master:master
```
